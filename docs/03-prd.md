# PRD: 학습 여정 대시보드

## 제품 개요

미래 의사과학자 챌린지 트랙 학습 여정 대시보드는 학생참여자의 학습 진행, 산출물, 멘토링, 피드백, 평가, 학습성과를 연결하여 2026년 1기 운영을 지원하고 이후 기수 반복 운영을 가능하게 하는 웹 기반 시스템이다.

## 문제 정의

기존 운영 방식에서는 학습 자료, 제출물, 멘토링 기록, 피드백, 평가, 성과 데이터가 분산되기 쉽다. 이로 인해 운영자는 위험 신호를 늦게 발견하고, 멘토는 산출물 맥락을 놓치며, PI는 프로그램 성과와 품질을 한눈에 보기 어렵다.

## 목표

- 학생에게 오늘/이번 주 해야 할 일을 명확히 보여준다.
- 운영자에게 즉시 조치 가능한 위험 신호를 제공한다.
- 멘토에게 담당 학생/팀의 산출물과 피드백 맥락을 제공한다.
- PI에게 산출물 품질과 프로그램 성과를 요약한다.
- 총괄 관리자에게 권한, 로그, 설정, 기수 반복 운영 기반을 제공한다.

## 주요 사용자 시나리오

### 학생참여자

학생은 모바일 웹에서 로그인 후 이번 주 할 일, 마감 임박 학습피스, 제출 필요 산출물, 예정된 멘토링, 받은 피드백을 확인한다.

### 프로그램운영자

운영자는 2026년 기수 대시보드에서 학습피스 지연, 산출물 미제출, 멘토링 불참/미배정 대상을 확인하고 리마인드 후보를 검토한다.

### 멘토

멘토는 담당 학생 또는 팀의 산출물과 학습 여정을 확인하고, 멘토링 기록과 산출물 피드백을 남기며 필요 시 루브릭 평가를 수행한다.

### PI / 연구책임자

PI는 산출물 품질, 루브릭 평가 결과, 학습성과 달성 현황, 고위험 학생/팀 요약을 확인한다.

### 시스템 총괄 관리자

총괄 관리자는 사용자, 역할, scope, 감사 로그, 접속 로그, PWA 설정, 전체 시스템 설정을 관리한다.

## 핵심 기능

- 역할 및 scope 기반 접근 제어
- 학생 개인 기준 학습 여정
- 모듈-콘텐츠-학습피스 객체 관리
- 유형별 학습피스 완료 기준
- 개인/팀 산출물 관리
- 멘토링 일정 및 기록
- 산출물 피드백과 루브릭 평가
- 위험군 및 리마인드 후보 추천
- 외부 설문 링크와 응답 여부 추적
- 감사 로그와 접속 로그
- 기본 PWA와 모바일 최적화

## 성공 기준

- P0 화면 27개가 구현되어 2026년 7월 운영 시작 전 데모 가능하다.
- 학생, 운영자, 멘토, PI, 총괄 관리자 역할별 핵심 플로우가 검증된다.
- 프로덕션 빌드가 통과한다.
- 50명 이하, 트랙 1개 운영 기준에서 수동 운영보다 위험 신호 확인 시간이 줄어든다.
