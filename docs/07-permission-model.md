# Permission Model

## 기본 원칙

권한은 역할(role)만으로 판단하지 않는다. 모든 권한은 역할과 접근 범위(scope)를 함께 판단한다.

```text
permission = role + scope + action + target
```

## 역할

- 학생참여자
- 프로그램운영자
- 연구책임자 / PI
- 멘토
- 시스템 총괄 관리자

## Scope 유형

- 전체 시스템
- 프로그램
- 기수
- 트랙
- 팀
- 학생

## 기본 접근 규칙

| 역할 | 기본 접근 범위 |
| --- | --- |
| 학생참여자 | 본인 데이터 |
| 멘토 | 담당 학생 / 담당 팀 |
| 프로그램운영자 | 담당 기수 |
| PI / 연구책임자 | 담당 프로그램 또는 기수의 성과/품질 데이터 |
| 시스템 총괄 관리자 | 전체 시스템 |

## 예시

| 사용자 | 역할 | Scope |
| --- | --- | --- |
| 학생 A | 학생참여자 | 2026 기수 / 본인 |
| 멘토 B | 멘토 | 2026 기수 / 2팀 |
| 운영자 C | 프로그램운영자 | 2026 기수 |
| PI D | 연구책임자 | 미래 의사과학자 챌린지 전체 |
| 총괄 E | 시스템 총괄 관리자 | 전체 시스템 |

## 접근 제어 요구사항

- 학생은 다른 학생의 산출물, 피드백, 평가를 볼 수 없어야 한다.
- 멘토는 담당 범위 밖 학생/팀 데이터를 볼 수 없어야 한다.
- 운영자는 담당 기수 밖 운영 데이터를 볼 수 없어야 한다.
- PI는 성과/품질 요약을 볼 수 있으나 민감한 시스템 설정은 볼 수 없어야 한다.
- 감사 로그, 접속 로그, 시스템 설정은 시스템 총괄 관리자만 관리한다.
