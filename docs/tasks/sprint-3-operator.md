# Sprint 3 Task: 운영자 화면

## 목표

프로그램운영자가 2026년 기수 운영을 관리하고 위험 신호를 조치할 수 있게 한다.

## 포함 범위

- 2026년 기수 대시보드
- 기수 상세
- 참여자 관리
- 팀 관리
- 멘토 배정
- 위험군 / 리마인드 추천
- 공지사항 목록 / 작성
- 리마인드 발송 이력

## 주요 데이터

- Cohort
- User
- Team
- RoleAssignment
- RiskSignal
- ReminderCandidate
- Notice

## 완료 기준

- 운영자는 학습피스 지연, 산출물 미제출, 멘토링 이슈를 볼 수 있다.
- 리마인드 후보는 자동 추천되지만 발송은 수동 기록으로 처리한다.
- 공지는 대상 역할과 scope를 지정할 수 있다.

## 검증

- 운영자는 담당 기수 밖 데이터를 볼 수 없다.
- 리마인드 후보가 위험 신호와 연결된다.
- 공지 작성/수정이 감사 로그에 남는다.
- 프로덕션 빌드가 통과한다.
