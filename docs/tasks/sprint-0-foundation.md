# Sprint 0 Task: 시스템 뼈대 / 운영 기반

## 목표

역할별 접근, 기본 기수 데이터, 로그, 시스템 설정, PWA 기본 설정을 갖춘 운영 가능한 뼈대를 만든다.

## 포함 범위

- 로그인 / 역할 선택
- 역할별 홈 라우팅
- 공통 레이아웃
- 권한 + scope 기본 구조
- 2026년 기수 seed 데이터
- 감사 로그
- 접속 로그
- 전체 시스템 설정
- PWA manifest, 앱 아이콘, 설치 가능 상태
- PWA 설정 화면

## 주요 데이터

- User
- RoleAssignment
- Cohort
- AuditLog
- AccessLog
- SystemSetting
- PwaSetting

## 권한

- 학생은 본인 홈만 접근한다.
- 멘토는 담당 학생/팀 범위만 접근한다.
- 운영자는 담당 기수 범위만 접근한다.
- PI는 성과/품질 요약 범위만 접근한다.
- 총괄 관리자는 시스템 설정과 로그에 접근한다.

## 검증

- 역할별 로그인 후 첫 화면이 다르게 표시된다.
- scope 밖 데이터 접근이 차단된다.
- 로그인 이벤트가 접속 로그에 남는다.
- 설정 변경이 감사 로그에 남는다.
- PWA manifest가 유효하다.
- 프로덕션 빌드가 통과한다.
