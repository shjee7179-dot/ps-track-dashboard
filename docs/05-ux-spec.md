# UX Spec

## 공통 UX 원칙

- 학생 화면은 할 일 중심으로 구성하고, 성장 여정은 보조 축으로 제공한다.
- 운영자 화면은 즉시 조치 가능한 위험 신호와 업무 큐를 우선한다.
- 멘토 화면은 담당 학생/팀, 산출물, 피드백, 멘토링 기록을 빠르게 오가게 한다.
- PI 화면은 운영 세부보다 성과와 품질을 우선한다.
- 총괄 관리자 화면은 권한, 로그, 설정을 빠르게 점검하게 한다.
- 모바일 웹에서 주요 학생 화면은 첫 화면에 핵심 행동이 보여야 한다.

## P0 화면별 핵심 컴포넌트

| 화면 | 주요 사용자 | 핵심 컴포넌트 |
| --- | --- | --- |
| 로그인 / 역할 선택 | 전체 | 로그인 폼, 역할 선택, 접근 범위 표시 |
| 학생 대시보드 | 학생 | 이번 주 할 일, 마감 임박, 제출 필요, 멘토링 일정, 피드백 |
| 2026년 기수 대시보드 | 운영자 | 운영 요약, 위험 신호, 리마인드 후보, 진행률 |
| 기수 상세 | 운영자 | 기수 정보, 일정, 참여자, 팀, 멘토 배정 |
| 참여자 목록 | 운영자 | 검색, 필터, 상태, 팀, 멘토 |
| 학생 상세 | 운영자/멘토 | 현재 할 일, 타임라인, 제출물, 피드백, 위험 신호 |
| 팀 목록 / 팀 상세 | 운영자/멘토 | 팀 구성, 담당 멘토, 팀 산출물, 팀 여정 |
| 멘토 배정 | 운영자 | 멘토 목록, 학생/팀 배정, 미배정 대상 |
| 모듈 목록 / 상세 | 운영자 | 모듈 정보, 콘텐츠, 학습피스, 일정 |
| 콘텐츠 목록 / 상세 | 운영자 | 콘텐츠 유형, 파일/링크, 연결 학습피스 |
| 학습피스 목록 / 상세 | 전체 | 유형, 완료 기준, 마감, 상태, 연결 성과 |
| 학생별 학습 여정 | 전체 권한별 | 타임라인, 상태, 필터, 지연 항목 |
| 학습피스 상태 관리 | 운영자/멘토 | 상태 변경, 사유, 로그 |
| 산출물 목록 | 전체 권한별 | 개인/팀 구분, 상태, 제출일, 평가 상태 |
| 산출물 제출 / 상세 | 학생/멘토 | 파일/링크, 버전, 피드백, 상태 |
| 산출물 리뷰 | 멘토/PI | 코멘트, 수정 요청, 연결 멘토링 |
| 루브릭 평가 | 멘토/PI | 항목별 점수, 총점, 종합 코멘트 |
| 멘토링 일정 | 학생/멘토/운영자 | 일정, 대상, 상태, 외부 링크 |
| 멘토링 기록 | 멘토 | 메모, 연결 산출물, 후속 액션 |
| 위험군 / 리마인드 추천 | 운영자 | 위험 유형, 심각도, 조치 상태 |
| 공지사항 목록 / 작성 | 운영자/전체 | 대상 범위, 제목, 내용, 읽음 여부 |
| 사용자 관리 | 총괄 | 사용자 목록, 상태, 역할 |
| 역할 / 접근 범위 관리 | 총괄 | role assignment, scope, 기간 |
| 감사 로그 | 총괄 | 이벤트, 사용자, 대상, 변경 전후 |
| 접속 로그 | 총괄 | 로그인, 접속 시각, IP/디바이스 |
| PWA 설정 화면 | 총괄 | 앱명, 아이콘, manifest 상태, 설치 안내 |
| 전체 시스템 설정 | 총괄 | 기본 기수, 알림, 파일 제한, 운영 설정 |

## 빈 상태

- 학생에게 할 일이 없으면 "이번 주 예정된 필수 학습이 없습니다"를 표시한다.
- 운영자에게 위험 신호가 없으면 "현재 조치가 필요한 위험 신호가 없습니다"를 표시한다.
- 멘토에게 담당 학생이 없으면 운영자에게 배정 요청 안내를 표시한다.

## 오류 상태

- 권한이 없는 데이터 접근 시 접근 불가 안내를 표시한다.
- 제출 파일 또는 링크 저장 실패 시 재시도 안내와 임시 저장 상태를 표시한다.
- 외부 설문 링크가 없으면 운영자에게 설정 필요 상태를 표시한다.
