# 미래 의사과학자 챌린지 트랙 학습 여정 대시보드 Product Frame

## 목적

미래 의사과학자 챌린지 트랙 학습 여정 대시보드는 2026년 1기 운영을 시작점으로, 이후 기수별 반복 운영이 가능한 객체 중심 운영 시스템을 만든다.

이 시스템은 단순 진도율 확인 도구가 아니라 학생 개인의 학습 여정, 산출물, 멘토링, 피드백, 평가, 학습성과를 하나의 흐름으로 연결하는 운영 플랫폼이다.

## 핵심 사용자

| 사용자 | 핵심 관심사 |
| --- | --- |
| 학생참여자 | 이번 주 해야 할 일, 산출물 제출, 피드백 확인, 나의 성장 여정 |
| 프로그램운영자 | 기수 운영 현황, 미완료/미제출/멘토링 이슈, 리마인드 대상 |
| 연구책임자 및 PI | 산출물 품질, 루브릭 평가, 학습성과, 프로그램 성과 지표 |
| 멘토 | 담당 학생/팀의 진행 상황, 산출물 리뷰, 멘토링 기록, 후속 액션 |
| 시스템 총괄 관리자 | 사용자, 역할, 접근 범위, 시스템 설정, 로그, 기수 반복 운영 |

## 운영 맥락

2026년도 프로그램은 6월 협약 이후 7월부터 10월까지 운영된다.

| 시기 | 운영 초점 |
| --- | --- |
| 6월 | 협약, 기수 생성, 참여자 등록, 멘토 배정, 학습 객체 확정 |
| 7월 | 오리엔테이션, 기초 모듈, 초기 학습피스, 사전 설문 |
| 8월 | 심화 학습, 멘토링, 중간 산출물 |
| 9월 | 프로젝트 수행, 피드백 반복, 산출물 고도화 |
| 10월 | 최종 산출물, 평가, 사후 설문, 기수 종료 리포트 |

## 제품 원칙

- 학생 개인 기준의 학습 여정을 기본 축으로 둔다.
- 모듈, 콘텐츠, 학습피스, 산출물, 성과는 객체로 관리한다.
- 객체와 일정은 기수별 인스턴스와 템플릿을 분리한다.
- 권한은 역할만이 아니라 접근 범위(scope)를 함께 적용한다.
- 1차는 트랙 1개, 50명 이하 운영에 최적화하되 이후 확장을 막지 않는다.
- 바이브코딩 개발을 전제로 문서는 얇게 만들고, 코드와 함께 갱신한다.

## 1차 성공 기준

- 학생은 본인의 할 일, 학습피스, 산출물, 멘토링, 피드백을 모바일에서 확인할 수 있다.
- 운영자는 2026년 기수의 위험 신호와 리마인드 대상을 확인할 수 있다.
- 멘토는 담당 학생/팀의 산출물을 리뷰하고 멘토링 기록을 남길 수 있다.
- PI는 산출물 품질, 평가 결과, 학습성과 요약을 확인할 수 있다.
- 총괄 관리자는 사용자, 역할, scope, 로그, 시스템 설정을 관리할 수 있다.
- 프로덕션 빌드가 통과하고 핵심 플로우가 수동 검증된다.
