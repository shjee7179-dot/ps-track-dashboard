# Development Roadmap

## 진행 방식

이 프로젝트는 바이브코딩을 전제로 진행한다. 문서는 개발을 지연시키는 완성본이 아니라, 개발 중 AI와 사람이 같은 기준을 보게 하는 얇은 기준 문서로 유지한다.

진행 순서:

```text
IA 확정
→ MVP 화면 확정
→ PRD 초안
→ SRS 초안
→ 데이터 모델
→ Sprint Task
→ 구현
→ 검증
→ 문서 갱신
```

## Sprint 0: 시스템 뼈대 / 운영 기반

- 로그인 / 역할별 라우팅
- 공통 레이아웃
- 권한 + scope 기본 구조
- 2026년 기수 seed 데이터
- 감사 로그
- 접속 로그
- 전체 시스템 설정
- PWA manifest / 앱 아이콘 / 설치 가능 상태
- PWA 설정 화면

## Sprint 1: 학생 여정 중심 MVP

- 학생 대시보드
- 학생 상세
- 학습피스 목록 / 상세
- 학습피스 상태
- 학생별 학습 여정

## Sprint 2: 산출물과 멘토링

- 산출물 제출 / 상세
- 산출물 리뷰
- 멘토링 일정
- 멘토링 기록
- 피드백 이력

## Sprint 3: 운영자 화면

- 기수 대시보드
- 참여자 관리
- 팀 관리
- 멘토 배정
- 위험군 / 리마인드 추천
- 공지 작성

## Sprint 4: 평가와 성과

- 루브릭 평가
- 학습성과 목록
- 학습성과 매핑
- 학생별 성과 증거
- PI 대시보드

## Sprint 5: 템플릿 / 리포트 / 모바일 품질

- 프로그램 템플릿
- 상대 일정 템플릿
- 설문 링크 / 응답 여부
- CSV 내보내기
- 모바일 최적화 점검
- PWA 품질 점검

## 검증 원칙

빌드, 런타임, 테스트 실패를 발견하면 재현, 가설, 검증, 빌드 확인 순서로 처리한다. 타입체크 또는 lint 통과만으로 완료하지 않고 프로덕션 빌드를 확인한다.
