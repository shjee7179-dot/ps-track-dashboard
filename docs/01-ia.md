# 1차 정보구조 IA

## 설계 기준

1차 IA는 학생 개인의 학습 여정을 기본 축으로 둔다. 객체와 일정은 필터 또는 연결 정보로 붙인다.

```text
프로그램 기수
  └─ 트랙 / 과정
      └─ 모듈
          └─ 콘텐츠
              └─ 학습피스
                  └─ 활동 / 제출 / 피드백 / 평가
                      └─ 학습성과 / 산출물
```

## 상위 메뉴

```text
홈 / 대시보드
├─ 학생 대시보드
├─ 운영자 대시보드
├─ 멘토 대시보드
├─ PI / 연구책임자 대시보드
└─ 총괄 관리자 대시보드

기수 관리
├─ 기수 생성
├─ 템플릿에서 기수 생성
├─ 2026년 기수
├─ 실제 일정 확정
├─ 참여자 관리
├─ 팀 관리
├─ 멘토 배정
└─ 다음 기수 복제

학습 객체 관리
├─ 프로그램 템플릿
├─ 모듈
├─ 콘텐츠
├─ 학습피스
├─ 활동 유형
├─ 학습성과
├─ 평가 루브릭
├─ 상대 일정 템플릿
└─ 객체 템플릿

학습 여정
├─ 학생별 여정
├─ 학생 상세
├─ 팀별 여정
├─ 모듈 / 콘텐츠 / 학습피스 필터
├─ 월 / 주차별 일정 필터
├─ 활동 로그
├─ 콘텐츠 열람 기록
└─ 위험군 / 리마인드 대상

산출물 / 성과
├─ 산출물 제출
├─ 개인 산출물
├─ 팀 산출물
├─ 산출물 리뷰
├─ 루브릭 평가
├─ 학습성과 매핑
├─ 학생별 성과 증거
├─ 사전 / 사후 설문 링크
├─ 설문 응답 여부
└─ 성과 리포트

멘토링 / 피드백
├─ 멘토링 일정
├─ 멘토링 기록
├─ 학생별 피드백
├─ 팀별 피드백
├─ 산출물 리뷰
├─ 후속 액션
├─ 루브릭 평가
└─ 멘토 활동 현황

공지 / 커뮤니케이션
├─ 공지사항
├─ 대상별 공지
├─ 알림 발송
├─ 리마인드 대상 추천
├─ 리마인드 발송 이력
└─ 메시지 템플릿

리포트
├─ 참여율 리포트
├─ 완료율 리포트
├─ 산출물 현황
├─ 멘토링 현황
├─ 학생 성장 리포트
├─ 기수 종료 보고서
└─ CSV 내보내기

시스템 관리
├─ 사용자 관리
├─ 역할 관리
├─ 접근 범위 관리
├─ 역할 배정
├─ 코드 / 분류값
├─ 감사 로그
├─ 접속 로그
├─ PWA 설정
└─ 시스템 설정
```

## 역할별 첫 화면

| 역할 | 첫 화면 |
| --- | --- |
| 학생참여자 | 학생 대시보드 |
| 프로그램운영자 | 2026년 기수 운영 대시보드 |
| 멘토 | 담당 학생/팀 대시보드 |
| PI / 연구책임자 | 성과 및 품질 대시보드 |
| 시스템 총괄 관리자 | 시스템 운영 대시보드 |

## 핵심 판단

- 학습 여정은 학생 개인 기준을 기본으로 한다.
- 학습피스 완료 기준은 유형별로 다르게 둔다.
- 산출물은 개인/팀 모두 가능하고 여러 개를 허용한다.
- 멘토링은 일정, 기록, 산출물 피드백, 루브릭 평가까지 연결한다.
- 운영 위험 신호는 학습피스 지연, 산출물 미제출, 멘토링 불참/미배정이다.
- 설문은 1차에서 외부 설문 링크와 응답 여부 추적으로 처리한다.
- 공지는 콘텐츠와 분리한다.
